01 — 프로젝트 개요

무엇을 만드는가

1~2인 인디 개발팀이 Unity + Go 스택으로 제작하는 밈 기반 미니게임 플랫폼. 오리지널 게임으로 시작해 UGC 생태계로 확장하는 넷플릭스식 구조를 지향한다.

🎮

장르

WarioWare식 자동 전환 미니게임 컬렉션. 2~3분 세션, 병맛/밈 정체성.

📺

플랫폼 모델

넷플릭스식. 오리지널로 품질 기준 수립, UGC로 콘텐츠 볼륨 확보.

📱

출시 플랫폼

Android 선행 출시 → iOS → Steam 패키지. 모바일과 PC UI 분리 설계.

⚙️

개발 환경

Unity + Claude MCP 워크플로우. Go 백엔드, k3s 인프라.

네이밍 배경: 2000년대 무료게임타운(무게타)의 정신을 현대적으로 계승. 줄임말 "숏게타"가 자연스럽게 밈화되는 것을 의도. 무게타 세대(90년대생)와 밈 소비 세대(2000년대생) 양쪽을 모두 포섭.

06 — 바이럴 루프 v1.3 신규

9:16 하이라이트 자동 공유

9:16 포맷이 TikTok·릴스와 완전히 일치한다는 것은 단순한 UI 선택이 아니다. 클리어 순간을 자동 캡처해 한 탭으로 공유하면 유저가 마케터가 된다.

핵심 인사이트: 게임 내 바이럴 루프를 만들지 않으면 소셜 공유는 유저의 수고에 의존한다. 자동화된 공유 경험이 "이게 뭐야 ㅋㅋ" → TikTok 업로드 → 신규 유입 루프를 닫는다.
🎬

하이라이트 자동 녹화

게임 클리어·실패 순간 직전 3초를 자동 캡처. 별도 설정 없이 항상 대기 상태. 9:16 비율이므로 TikTok·릴스·YouTube Shorts에 별도 편집 없이 업로드 가능.

📤

원탭 공유 플로우

결과 화면에서 공유 버튼 하나로 영상 + 점수 + 숏게타 워터마크 자동 합성. 카카오톡·인스타그램·TikTok 직접 연동. 공유 시 리더보드 보너스 포인트 지급.

🌊

바이럴 증폭 구조

07 — 추천 알고리즘

태그 기반 큐레이션 + DDA

유저 플레이 패턴을 태그로 분석해 취향 맞춤 세션을 구성하고, 유저 성공률(Success Rate) 기반의 다이나믹 난이도 조절(DDA)을 결합해 이탈률을 낮춘다.

장르반응속도타이밍기억력눈치운빨집중력멀티태스킹
강도 (DDA)하드코어중간잔잔자극적
소재동물음식직장연애공포일상판타지
밈도병맛순수개그블랙유머레트로인터넷밈
단계동작데이터 소스비고
STEP 01 이력 수집완료 / 이탈 / 재도전 + 성공률(SR) 기록Analytics API세션 단위 기록
STEP 02 태그 가중치플레이 패턴 → 태그별 선호도 스코어Game Registry + 이력재도전 횟수 가중 2배
STEP 03 DDA 강도 조절SR > 80% → 강도↑ / SR < 40% → 강도↓실시간 SR 계산강도 태그를 동적 재가중
STEP 04 세션 큐 구성선호 태그 비중↑ + DDA 강도 + 신규 발견 20%추천 엔진신규 발견 비율 고정
STEP 05 피드백 반영클리어율·재도전율·이탈 시점으로 업데이트실시간 이벤트세션 종료 후 배치

📈 DDA 작동 원리

Success Rate = 최근 10게임 클리어 수 / 10. SR 임계값 벗어나면 다음 세션 큐에서 "강도" 태그 가중치 자동 조정. 실력이 늘면 자연스럽게 더 어려운 게임이 등장.

🎯 이탈률 감소 효과

너무 쉬우면 지루해서 이탈. 너무 어려우면 좌절해서 이탈. DDA는 유저를 "아 좀 아깝다" 구간에 계속 유지시킨다. 리텐션 향상의 핵심 메커니즘.

⚠️ 설계 주의사항

DDA는 유저에게 보이지 않아야 한다. 성취감이 깨지지 않도록. 강도 조절폭을 ±1단계로 제한해 급격한 변화 방지.

Cold Start: 초기 3세션은 오리지널 게임 + 인기 순 + DDA 중간 강도 기본값. 3세션 이후부터 태그 + DDA 동시 적용.
07 — 경쟁 시스템

게임별 리더보드

글로벌 랭킹 하나만 있으면 상위 0.1%만 경쟁한다. 게임별 리더보드로 누구나 특정 게임의 랭커가 될 수 있는 입체적 경쟁 구조.

클라이언트
Unity (2022.3 LTS)
Steam + Android + iOS 멀티 타겟. Addressables로 게임 번들 런타임 로드. UGC 게임을 동일 인터페이스로 실행.
AI 개발 워크플로우
Claude Desktop + MCP for Unity
CoplayDev unity-mcp 패키지. Claude가 Unity Editor 직접 조작. 씬 생성, 스크립트 작성, 컴포넌트 수정 자동화.
백엔드 API
Go
Game Registry / Ranking / Analytics / Creator API. 게임 서버 경험 직접 활용. HMAC 기반 Anti-cheat 구조 적용.
인프라
k3s + ArgoCD
기존 계획 스택 그대로 적용. GitHub Actions CI/CD. Traefik 인그레스. Sealed Secrets.
게임 번들 배포
Unity Addressables + CDN
게임 번들 런타임 다운로드. UGC 게임 등록 시 CDN 업로드 자동화. 플랫폼 독립적 로드 구조.
핵심 인터페이스
IMinigame (C#)
모든 미니게임(오리지널+UGC)이 구현하는 표준 계약. GameId / Tags / OnGameStart / GetScore / OnInput 포함.
C# · UnityIMinigame — UGC 확장성의 핵심 인터페이스
// 모든 미니게임이 구현해야 하는 표준 계약
public interface IMinigame
{
    string   GameId    { get; }  // "frog_catch_v1"
    string   Title     { get; }  // "개구리 잡아라"
    string   CreatorId { get; }  // 오리지널 or UGC 크리에이터
    float    TimeLimit { get; }  // 제한시간 (초)
    string[] Tags      { get; }  // ["반응속도", "동물", "병맛"]

    void OnGameStart();
    void OnGameEnd();
    int  GetScore();
    void OnInput(InputEvent input);  // 터치/클릭 추상화
}
MCP Prompt Engineering 가이드 필요 v1.3: AI가 생성한 코드의 일관성·유지보수성을 위해 내부 가이드를 별도 문서로 관리. 핵심 항목 — C# 네이밍 컨벤션 (파스칼 케이스 강제), 폴더 구조 규칙 (Scripts/Minigames/), IMinigame 구현 체크리스트, 주석 형식 표준, 테스트 코드 요건. Claude에게 매번 전달하는 시스템 프롬프트 형태로 운용. 가이드 없이 AI 생성 코드를 누적하면 6개월 후 레거시가 된다.
